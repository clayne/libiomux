Detailed Description

I/O multiplexer.

Data Structure Documentation

struct iomux_callbacks_t

iomux callbacks structure

Data Fields
iomux_input_callback_t mux_input The callback called when there is new data on the monitored fd.
iomux_output_callback_t mux_output If not NULL, it will be called when it's possible to write new data on fd.
iomux_timeout_callback_t mux_timeout If not NULL, it will be called when a timeout on fd expires.
iomux_eof_callback_t mux_eof If not NULL, it will be called when EOF is reached and the fd can be closed.
iomux_connection_callback_t mux_connection If not NULL and fd is a listening socket, it will be called when a new connection is accepted on fd.
iomux_free_data_callback_t mux_free_data If not NULL this callback will be used instead of free() when an output chunk should be released.
void * priv A pointer to private data which will be passed to all the callbacks as last argument.

Handle input coming from a managed filedescriptor.

Parameters
iomuxThe iomux handle
fdThe fd the timer relates to
datathe data read from the filedescriptor
lenthe size of the data being provided
privthe private pointer registered with the callbacks
Returns
The number of bytes actually processed by the receiver If less than 'len' bytes have been processes (because underrun or similar) the remaining data will be kept by the iomux and provided back at next call (stopping reading from the filedescriptor if necessary)
typedef iomux_output_mode_t(* iomux_output_callback_t)(iomux_t *iomux, int fd, unsigned char **data, int *len, void *priv)

Callback called to poll for data to be written to a managed filedescriptor.

Parameters
iomuxThe iomux handle
fdThe fd the timer relates to
dataA reference to the pointer to where the data is stored
lenA pointer to where to store length of the data
privthe private pointer registered with the callbacks
Returns
the iomux_output_mode which determines if the data has to be copied, freed or ignored (in which case the caller needs to take care of releasing the underlying memory)
Note
If no data is available for writing and hence has been copied to the data pointer, the callback MUST ensure setting *len to zero so that the iomux doesn't try sending garbage data

Register timed callback.

Parameters
iomuxThe iomux handle
timeoutThe timeout to schedule
cbThe callback to call when the timeout expires
privA private context which will be passed to the callback
free_ctx_cbAn optional callback which, if provided, will be called when the timeout is being destroyed.
This can be useful to catch timeouts which are eventually removed before being fired.
Note
The free_ctx_cb, if provided, will be called also in the case the timeout is correctly fired. The caller should wait until the free_ctx_cb is called before releasing the resources eventually referenced by the priv pointer.
Returns
The timeout id on success; 0 otherwise.
